# Tableau Public Dashboard: Step-by-Step Guide

This guide walks you through building the complete Cohort & Retention dashboard in Tableau Public from the CSV exports generated by `analysis.ipynb`.

**IMPORTANT CONCEPTS** before you start:
- **Blue pills = Dimensions** (categorical, discrete). They create headers/rows/columns.
- **Green pills = Measures** (continuous numbers). They get aggregated (SUM, AVG, etc.).
- When you drag a number field onto Rows/Columns, Tableau defaults to SUM. You almost always need to change this.
- **Right-click drag** gives you more control than left-click drag — it lets you choose the aggregation and dimension/measure type immediately.

---

## Prerequisites

1. Run `analysis.ipynb` (Kernel → Restart & Run All) to generate the 4 CSV files
2. Download [Tableau Public](https://public.tableau.com/en-us/s/download) (free)
3. You should have these files in your `project-c-cohort-retention/` folder:
   - `tableau_cohort_retention.csv`
   - `tableau_rfm_customers.csv`
   - `tableau_ltv_data.csv`
   - `tableau_churn_risk.csv`

## Part 1: Connect Your Data

We'll connect each CSV as a **separate data source** so they don't get joined together.

### Data Source 1: Cohort Retention
1. Open **Tableau Public**
2. Under "Connect" on the start screen → click **Text file**
3. Navigate to your `project-c-cohort-retention/` folder → select `tableau_cohort_retention.csv`
4. The Data Source tab opens showing a preview of the data. Verify you see columns: `cohort`, `cohort_period`, `customers`, `cohort_size`, `retention_rate`
5. **CRITICAL:** Before building any sheets, fix the data types:
   - Click the `#` icon above `Cohort Period` → change to **Dimension** (not a measure). To do this: in the Data pane on the left side of a new worksheet, right-click `Cohort Period` → select **Convert to Dimension**
   - `Cohort` should already be a dimension (blue Abc icon). If it shows as a measure, right-click → Convert to Dimension

### Data Sources 2–4 (add later)
- After building Sheet 1, go to **Data** menu (top menu bar) → **New Data Source** → **Text file** → select the next CSV
- Repeat for each CSV. You'll switch between data sources by clicking the data source name in the Data pane dropdown

### SHEET 1: Cohort Retention Heatmap

**Data source:** `tableau_cohort_retention`

**Goal:** A grid where rows = cohorts, columns = months since first purchase, squares colored by retention %.

**Step by step:**

1. **Create a new worksheet** (click the tab with a + icon at the bottom). Right-click the tab → Rename → type `Cohort Retention Heatmap`

2. **Set up Rows — Cohort:**
   - In the Data pane on the left, find `Cohort` (it should have a blue "Abc" icon)
   - Drag `Cohort` to the **Rows** shelf (the horizontal bar that says "Rows" at the top)
   - You should see a list of cohort months (2016-09, 2016-10, etc.) appear on the left side of the view

3. **Set up Columns — Cohort Period:**
   - Find `Cohort Period` in the Data pane
   - **IMPORTANT:** If it has a green `#` icon, it's currently a Measure. Right-click it in the Data pane → select **Convert to Dimension**. It should now show a blue `#` icon
   - Drag `Cohort Period` to the **Columns** shelf
   - You should now see numbers (0, 1, 2, 3, 4, ...) across the top of the view
   - If you only see a single axis with one number, the field is still being treated as a continuous measure. Right-click the pill on the Columns shelf → make sure **Dimension** is selected (not Measure), and **Discrete** is selected (not Continuous)

4. **Set the Mark Type:**
   - In the **Marks** card (left side, below the Data pane), click the dropdown that probably says "Automatic" → change it to **Square**

5. **Add Color — Retention Rate:**
   - Drag `Retention Rate` to the **Color** icon on the Marks card
   - Tableau will default to `SUM(Retention Rate)`. Since each cohort+period combination has exactly one row, SUM works. But to be safe: right-click the green pill that appeared on the Marks card → select **Average** instead of **Sum**
   - Click the **Color** legend that appeared on the right → click **Edit Colors**:
     - Palette: select **Red-Green Diverging**
     - Check **Reversed** (so green = high retention, red = low)
     - Click **Advanced** → check **Center** → set to `50`
     - Click OK

6. **Add Labels:**
   - Drag `Retention Rate` to the **Label** icon on the Marks card (the "T" icon)
   - Right-click this pill → select **Average** (same as Color)
   - Right-click the pill again → **Format** → in the pane that opens on the left, under "Default" → Numbers → click the dropdown → select **Number (Custom)** → set Decimal places to `1` → add `%` as a suffix
   - The squares should now show values like `100.0%`, `5.4%`, `0.3%`

7. **Filter to first 12 months (optional but recommended):**
   - Drag `Cohort Period` to the **Filters** shelf (the area that says "Filters" on the left)
   - Select "Range of values" → set max to `12` → OK

8. **Title:**
   - Double-click the title area at the top → type: `Cohort Retention Heatmap (% of Original Cohort)`

**What it should look like:** A colored grid. Top-left corner shows 100% for all cohorts (Month 0 = acquisition month). Colors rapidly shift to red/orange by Month 1 as retention drops. Later months are mostly empty (not enough time elapsed).

### SHEET 2: Average Retention Curve

**Data source:** `tableau_cohort_retention`

1. **New worksheet** → rename to `Average Retention Curve`

2. **Columns — Cohort Period:**
   - Drag `Cohort Period` to Columns
   - Make sure it's a **Discrete Dimension** (blue pill, not green). Right-click the pill → Dimension + Discrete if needed

3. **Rows — Retention Rate:**
   - Drag `Retention Rate` to Rows
   - Right-click the green pill → select **Average** (this averages across all cohorts for each period)

4. **Mark Type:**
   - Change mark type to **Line** (dropdown on the Marks card)

5. **Add Labels:**
   - Drag `Retention Rate` to **Label** → right-click → **Average**
   - Format to 1 decimal with `%` suffix (same as Sheet 1)

6. **Add markers to the line:**
   - On the Marks card, click **Color** → at the bottom, find "Markers" → select **All**

7. **Filter:**
   - Drag `Cohort Period` to Filters → Range → 0 to 12

8. **Reference Line (optional):**
   - Click the **Analytics** tab (next to Data tab on the left)
   - Drag **Average Line** onto the chart
   - This shows the overall average retention as a horizontal line

9. **Title:** `Average Retention Curve Across All Cohorts`

**What it should look like:** A line starting at 100% (Month 0), dropping sharply to low single digits by Month 1, then flattening near 0% for subsequent months.

## Part 3: Build the Dashboard

1. Click the **New Dashboard** icon at the bottom (looks like a grid with a + sign)
2. **Size:** On the left panel under "Size", click the dropdown → select **Automatic** (adapts to screen) or **Fixed** 1200×900
3. **Name:** Right-click the tab → rename to `Cohort & Retention Analytics`

**Layout — drag sheets from the left panel into the canvas:**

```
┌──────────────────────────────────────────────┐
│  [Text: Title bar with name + links]         │
├──────────────────────────────────────────────┤
│  Cohort Retention Heatmap (full width)       │
├───────────────────────┬──────────────────────┤
│  Average Retention    │  RFM Segment         │
│  Curve                │  Distribution        │
├───────────────────────┼──────────────────────┤
│  LTV by State/Segment │  Churn Risk          │
│                       │  Distribution        │
└───────────────────────┴──────────────────────┘
```

4. **Add title:** From the left panel under "Objects", drag **Text** to the very top of the dashboard → type:
   ```
   Olist E-Commerce: Cohort & Retention Analytics
   Tobi Bolu | LinkedIn | GitHub
   ```
   (You can't add real hyperlinks in dashboard text, but you can include the URLs as text)

5. **Add Filter Actions** (this makes the dashboard interactive — clicking one chart filters the others):
   - Go to **Dashboard** menu (top menu bar) → **Actions** → **Add Action** → **Filter**
   - **Action 1 — Segment Drill-Down:**
     - Name: `Filter by Segment`
     - Source Sheets: check `RFM Segments`
     - Target Sheets: check `LTV by Segment` and `Churn Risk Distribution`
     - Run action on: **Select**
     - Clearing the selection will: **Show all values**
     - Click OK
   - **Action 2 — State Drill-Down:**
     - Name: `Filter by State`
     - Source: `LTV by State`
     - Target: all other sheets
     - Run on: Select
   - **Action 3 — Risk Drill-Down:**
     - Name: `Filter by Risk Level`
     - Source: `Churn Risk Distribution`
     - Target: `RFM Segments`, `LTV by Segment`
     - Run on: Select

   Now clicking a segment bar filters LTV and churn. Clicking a state filters everything. Clicking a risk level filters segments and LTV.

## Part 4: Build the Story

A Story is a sequence of annotated "story points" — like a slide deck built from your sheets and dashboards.

1. Click the **New Story** icon at the bottom (looks like an open book with a + sign)
2. Set size to match your dashboard (Automatic or 1200×900)

### Story Point 1: "The Retention Problem"
- From the left panel, drag the `Cohort Retention Heatmap` sheet onto the canvas
- At the top, there's a grey caption bar — click it and type: `97% of customers never return after their first purchase. Retention drops below 5% by Month 1 across all cohorts.`
- **Add an annotation:** Right-click on the Month 1 column in the heatmap → select **Annotate** → **Area** → draw a box around the Month 1 column → type: `Sharp drop here — structural marketplace issue`

### Story Point 2: "Who Are Our Best Customers?"
- Click **Blank** at the top of the Story pane (or "New Blank Point")
- Drag the `RFM Segments` sheet onto the canvas
- Caption: `RFM segmentation reveals Champions and Loyal Customers are a tiny fraction of the base but drive outsized revenue.`
- Click **Duplicate** (the icon next to the caption) to create a copy of this story point
- On the duplicate, swap in the `Segment Profiles` sheet (drag it over the existing sheet) so viewers can flip between the bar chart and the profile table

### Story Point 3: "Where Is the Value?"
- New blank story point
- Drag `LTV by State` sheet
- Caption: `Remote states show the highest per-customer LTV — likely inflated by freight costs. Sao Paulo has the highest volume but lowest per-customer value.`

### Story Point 4: "Who's At Risk?"
- New blank story point
- Drag your full **Dashboard** (listed under "Dashboards" in the left panel — you can drag entire dashboards into story points)
- Caption: `"At Risk (Repeat)" customers are the highest-priority group — they've already demonstrated willingness to repurchase. Start here.`
- The dashboard's filter actions will work inside the story point, so viewers can click around

### Story Point 5: "Recommendations"
- New blank story point
- Drag a **Text** object from the left panel
- Double-click to edit and type:

```
Key Recommendations

1. Launch re-engagement campaigns targeting "At Risk (Repeat)" customers
   within the 50-day repurchase window identified in the retention analysis.

2. Invest differently by geography: high-touch outreach for high-LTV
   remote states (PB, AC, RO), automated volume campaigns for SP.

3. Build brand awareness at point of purchase — most customers don't realize
   they bought through Olist, which kills structural retention.

4. Set minimum review thresholds for sellers in high-churn states to
   protect the customer experience.

5. Target the 3% repeat buyer segment with loyalty incentives — they
   already spend 1.8x to 5.2x more than one-time buyers.
```

## Part 6: Creating Hierarchies (Drill-Down)

Hierarchies let recruiters click a `+` icon to drill into deeper detail.

### How to create a hierarchy:
1. In the **Data** pane on the left, find the field you want as the top level (e.g., `Segment`)
2. **Right-click** it → **Hierarchy** → **Create Hierarchy**
3. Name the hierarchy (e.g., "Customer Segments")
4. Now **drag** other fields onto the hierarchy name to add them as child levels — drag them IN ORDER (top to bottom = general to specific)
5. When you use the top-level field in a view, a small `+` icon appears on the pill — clicking it expands to the next level

### Hierarchies to create:

**In `tableau_cohort_retention` data source:**
| Hierarchy | Top Level | Child 1 | Child 2 |
|---|---|---|---|
| Cohort Detail | Cohort | Cohort Size | Cohort Period |

**In `tableau_rfm_customers` data source:**
| Hierarchy | Top Level | Child 1 | Child 2 | Child 3 |
|---|---|---|---|---|
| Customer Segments | Segment | R Score | F Score | M Score |

**In `tableau_ltv_data` data source:**
| Hierarchy | Top Level | Child 1 |
|---|---|---|
| Geography → Segment | State | Segment |

**In `tableau_churn_risk` data source:**
| Hierarchy | Top Level | Child 1 | Child 2 |
|---|---|---|---|
| Risk Funnel | Churn Risk | Segment | State |

## Part 7: Polish & Tips

**Color consistency across sheets:**
- When you assign colors for segments in Sheet 3, those colors carry across all sheets using the same data source
- For churn risk colors (different data source), manually match the colors

**Clean up the appearance:**
- Right-click any axis → **Format** → under "Lines", set Grid Lines to **None** for a cleaner look
- Right-click column/row headers you don't need → **Hide Field Labels for Columns/Rows**
- Remove "Sheet 1" labels from dashboard: click the sheet in the dashboard → find the title bar → right-click → **Hide Title**

**Tooltips (what shows on hover):**
- Click **Tooltip** on the Marks card → customize the text, e.g.:
  ```
  Cohort: <Cohort>
  Month: <Cohort Period>
  Retention: <AVG(Retention Rate)>%
  Original Size: <AVG(Cohort Size)>
  ```

**Mobile layout:**
- In the Dashboard tab → **Device Preview** (toolbar) → **Phone** → arrange a simplified mobile layout

**Story presentation tips:**
- Keep captions to 1-2 sentences — insight-driven, not descriptive
- "97% never return" beats "This chart shows retention rates by cohort"
- Use annotations sparingly to highlight the single most important thing on each story point
